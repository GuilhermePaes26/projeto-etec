FTPS, ou FTP Secure, é uma extensão do protocolo FTP (File Transfer Protocol) que adiciona uma camada de segurança por meio da criptografia. Ele é projetado para proteger a transferência de arquivos através de redes, garantindo que os dados e credenciais estejam protegidos contra interceptação e acesso não autorizado.

O que é FTPS?

FTPS é essencialmente o FTP com segurança adicional. Ele utiliza criptografia para proteger os dados e as credenciais durante a comunicação entre cliente e servidor.

FTPS usa protocolos SSL (Secure Sockets Layer) ou TLS (Transport Layer Security) para criptografar a comunicação. Essa criptografia garante a confidencialidade e integridade dos dados transmitidos.

Como Funciona o FTPS?

Estabelecimento da Conexão Segura: Porta de Controle: FTPS geralmente usa a mesma porta de controle (porta 21) do FTP, mas a comunicação é criptografada após o estabelecimento da conexão. Portas de Dados: As portas usadas para a transferência de dados podem variar e dependem do modo de operação (ativo ou passivo).

Modos de Conexão: Modo Ativo: O cliente abre uma porta para receber dados e o servidor se conecta a essa porta. No FTPS ativo, a comunicação de dados é criptografada. Modo Passivo: O servidor abre uma porta para a comunicação de dados e o cliente se conecta a essa porta. No FTPS passivo, a porta de dados também é criptografada.

Características do FTPS

  • Confidencialidade: Os dados transmitidos são criptografados, o que impede que sejam lidos por terceiros.
  • Integridade: A criptografia protege os dados contra alterações durante a transmissão.
  • Autenticação: O uso de certificados digitais para autenticar o servidor e, opcionalmente, o cliente, garantindo que as partes envolvidas na comunicação são quem afirmam ser.

Compatibilidade com FTPS:

  • FTPS é compatível com clientes e servidores FTP existentes, mas requer que ambos suportem SSL/TLS para fornecer segurança.
  • FTPS Explícito (Explicit FTPS): O cliente inicia a conexão FTP padrão e depois solicita a segurança (SSL/TLS) através de um comando. O servidor e o cliente estabelecem a criptografia antes da transmissão de dados.
  • FTPS Implícito (Implicit FTPS): A conexão é automaticamente criptografada desde o início. Geralmente usa uma porta diferente, como a 990, para a comunicação criptografada.

Comandos FTPS

Comando Descrição
USER Fornece o nome de usuário para autenticação no servidor.
PASS Fornece a senha para autenticação no servidor.
LIST Lista os arquivos e diretórios no diretório atual do servidor.
RETR Recupera (baixa) um arquivo do servidor para o cliente.
STOR Armazena (envia) um arquivo do cliente para o servidor.
PWD Mostra o diretório de trabalho atual no servidor.
CWD Muda o diretório de trabalho no servidor.
DELE Remove (deleta) um arquivo no servidor.
MKD Cria um novo diretório no servidor.
RMD Remove (deleta) um diretório no servidor.
ABOR Aborta a operação em andamento (por exemplo, uma transferência de arquivo).
QUIT Encerra a sessão FTP com o servidor.

Importância na Web

FTPS é crucial para garantir a segurança na transferência de arquivos na web, oferecendo criptografia e autenticação para proteger os dados e credenciais durante a comunicação entre cliente e servidor.
