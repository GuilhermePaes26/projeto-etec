File Transfer Protocol (Protocolo de Transferência de Arquivos), é um protocolo de rede usado para transferir arquivos entre um cliente e um servidor através de uma rede, como a Internet ou uma rede local. É uma das formas mais antigas e comuns de transferir arquivos e ainda é amplamente utilizado em diversas aplicações.

O que é FTP?

FTP é projetado especificamente para a transferência de arquivos. Permite o upload (envio) e download (recebimento) de arquivos entre um computador (cliente) e um servidor de arquivos.

FTP opera sobre o protocolo TCP (Transmission Control Protocol), garantindo que os dados sejam entregues de forma confiável e em ordem.

Como Funciona o FTP?

Portas: FTP usa duas portas TCP principais: a porta 21 para o controle da conexão e a porta 20 para a transferência de dados em modo ativo. No modo passivo, a porta para a transferência de dados é negociada dinamicamente.

Autenticação: O cliente se conecta ao servidor FTP na porta 21 e fornece um nome de usuário e senha para autenticação. Embora FTP suporte autenticação básica, não criptografa as credenciais e dados por padrão.

Modo Ativo: O cliente abre uma porta para receber dados e o servidor FTP se conecta a essa porta para transferir os dados.

Modo Passivo: O servidor FTP abre uma porta e o cliente se conecta a essa porta para a transferência de dados. O modo passivo é geralmente usado quando o cliente está atrás de um firewall.

Comandos: O cliente FTP envia comandos para o servidor (por exemplo, LIST para listar arquivos, RETR para recuperar arquivos, STOR para armazenar arquivos).

Respostas: O servidor FTP responde aos comandos com códigos de status e mensagens.

Características do FTP

  • Transferência de Arquivos: Permite a transferência de arquivos grandes e múltiplos arquivos simultaneamente e suporta diferentes tipos de arquivos e estruturas de diretórios.
  • Autenticação e Controle de Acesso: FTP pode ser configurado para permitir o acesso anônimo (sem autenticação) ou exigir credenciais de usuário e permite configurações detalhadas de permissões para usuários e grupos.
  • Modos de Transferência: Modo Binário: Transfere arquivos em formato binário, preservando a integridade dos dados e Modo ASCII: Transfere arquivos de texto, convertendo os dados para o formato de texto apropriado para a plataforma de destino.

Segurança no FTP

  • FTP Sem Segurança (FTP): As transmissões são feitas em texto claro, o que significa que credenciais e dados podem ser interceptados por terceiros.
  • FTP sobre SSL/TLS (FTPS): Adiciona uma camada de segurança ao FTP usando criptografia SSL/TLS para proteger a transmissão de dados.
  • SFTP (SSH File Transfer Protocol): Um protocolo diferente que usa o SSH (Secure Shell) para criptografar a transferência de arquivos e fornece maior segurança do que o FTP tradicional.

Comandos FTP mais comuns

Comandos Descrição
USER Fornece o nome de usuário para autenticação.
PASS Fornece a senha para autenticação.
LIST Lista os arquivos e diretórios no diretório atual.
RETR Recupera (baixa) um arquivo do servidor.
STOR Armazena (envia) um arquivo para o servidor.
PWD Mostra o diretório de trabalho atual no servidor.
CWD Muda o diretório de trabalho no servidor.
